API Reference

All REST API endpoints are mounted under /api/v1.

When the server is running, interactive API documentation is available at http://localhost:8000/docs (Swagger UI).

Health

GET /api/v1/health Check server health

Returns a simple health check response.

Response:

{ "status": "ok" }

Projects

POST /api/v1/projects Create a project

Creates a new project with a GitHub repository and optional source tokens.

Request body:

{ "repo_owner": "facebook", "repo_name": "react", "github_token": "ghp_...", "notion_token": "secret_...", "discord_token": "MTAx..." }
GET /api/v1/projects List all projects

Returns a list of all projects with their current status and metadata.

Response:

[ { "id": "uuid", "repo_owner": "facebook", "repo_name": "react", "dataset": "facebook/react", "status": "INDEXED", "last_synced_at": "2026-07-04T12:00:00Z" } ]
GET /api/v1/projects/{id} Get project details

Returns detailed information about a specific project.

DELETE /api/v1/projects/{id} Delete a project

Deletes a project and prunes its dataset from the knowledge graph via cognee.forget().

POST /api/v1/projects/{id}/authenticate Add or validate source token

Add a new API token for a source or validate that existing tokens have correct permissions.

Ingestion & Sync

POST /api/v1/projects/{id}/ingest Initial ingestion

Triggers a full initial ingestion for the project. Runs in the background.

Flow:

  1. Fetches data from all configured sources concurrently
  2. Deduplicates items by SHA-256 content hash
  3. Stages unique items via cognee.add()
  4. Runs cognee.cognify() to build the knowledge graph
POST /api/v1/projects/{id}/sync Incremental sync

Pulls only new or updated data since the last sync timestamp. Fast, incremental update.

GET /api/v1/projects/{id}/status Sync status

Returns the current sync status for a project.

Response:

{ "status": "INDEXED", "last_synced_at": "2026-07-04T12:00:00Z", "last_ingested_at": "2026-07-04T10:00:00Z" }

Chat & Conversations

GET /api/v1/projects/{id}/conversations List conversations

Lists all chat conversations for a project.

POST /api/v1/projects/{id}/conversations Create conversation

Creates a new chat conversation scoped to the project's dataset.

GET /api/v1/chat/{conversation_id}/messages Get messages

Returns all messages in a conversation.

POST /api/v1/chat/{conversation_id} Send message

Sends a message in a conversation. Triggers LLM response with knowledge graph context.

Request body:

{ "message": "Why was PR #54 rejected?" }

Files

POST /api/v1/files/{project_dataset}/upload Upload file

Upload a PDF, Markdown, or TXT file for ingestion into the project's dataset.

Content-Type: multipart/form-data

Error Handling

Errors are returned per-source (e.g. {"github": "ValueError: GITHUB_PAT_TOKEN is not configured."}), so one source failing doesn't block others.

Standard HTTP status codes are used:

  • 200 — Success
  • 201 — Created
  • 400 — Bad Request
  • 404 — Not Found
  • 500 — Internal Server Error
