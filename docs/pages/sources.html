Ingestion Sources

Every platform Repollect can ingest data from.

GitHub

Ingestor: app/ingest/github/github.py

The GitHub ingestor fetches data via the REST and GraphQL APIs. It intelligently filters files — skipping binaries, lock files, build artifacts, and vendor directories while prioritizing documentation and important source code.

ArtifactDetails
Repository filesREADME.md, CONTRIBUTING.md, CHANGELOG.md, LICENSE, docs/, src/
IssuesAll states (open/closed), with labels, comments, author
Pull RequestsAll states, with review comments, base/head branches
DiscussionsVia GraphQL API, with category tags
ReleasesRelease notes, tags, pre-release flag

Discord

Ingestor: app/ingest/discord/discord.py

The Discord ingestor uses a Bot token to read messages from servers (guilds). It intelligently skips noisy channels (random, memes, spam) and binary attachments.

ArtifactDetails
GuildsFiltered by allowed list; skips unavailable guilds
ChannelsSkips voice, stage, category channels; skips noisy channels (random, memes, spam)
ThreadsActive threads fetched for each guild
MessagesHistory via pagination; removes system messages and exact duplicates
AttachmentsPDF (text extraction via pypdf) and text files; skips binaries (.exe, .zip, .mp4, .mov)

Notion

Ingestor: app/ingest/notion/notion.py

The Notion ingestor uses an Integration Token to search workspaces and fetch pages. It recursively resolves nested blocks and renders 15+ block types to Markdown.

ArtifactDetails
PagesFull workspace search, recursive block fetching
DatabasesMetadata extraction
Blocks15+ block type renderers → Markdown (headings, lists, code, quotes, tables, to-do, callouts, equations)
Rich textBold, italic, code, strikethrough, links preserved
Nested pagesRecursive child page/database ingestion

Local Files

Ingestor: app/ingest/files/files.py

Upload documents directly via the REST API or UI. Supported formats:

  • PDF (text extraction via pypdf)
  • Markdown (.md)
  • Plain text (.txt)

Planned Sources

New sources follow the established pattern: a SourceIngestor class fetches data via HTTP API / SDK, produces list[DataItem], and the handler deduplicates + stages + cognifies.

Each new ingestor lives at app/ingest/<source>/<source>.py.

SourcePriorityDescription
Static Documentation SitesHighDocusaurus, Mintlify, GitBook, MkDocs — crawl and ingest all pages

Linear / JiraMediumIssues, epics, sprints, comments, project descriptions

ConfluenceMediumSpaces, pages, attachments, comments, labels
EmailLowEngineering mailing lists, RFC discussions, IMAP/Gmail API
RSS / Blog FeedsLowEngineering blogs, changelog RSS, release notes
