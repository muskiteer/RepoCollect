Overview

What is Repollect?

Repollect (powered by Cognee) is an onboarding knowledge layer for open-source projects and new hires. It solves the "Where's my context?" problem by ingesting your scattered project history—GitHub issues, Pull Requests, Discord threads, and Notion documents—into a unified, queryable store.

Why Cognee?

Standard Retrieval-Augmented Generation (RAG) relies on vector databases alone. But codebases and team communications are inherently relational. Cognee provides a hybrid Graph-Vector architecture.

By using Cognee, Repollect doesn't just find text that looks similar. It understands that "PR #482 fixed a bug in the auth-service, which was discussed by Alice in #incident-channel, and decided upon in a Notion meeting." Cognee's dual-layered approach (graph + vector) allows us to trace these relationships flawlessly and inject perfect context into the LLM prompt.

Pipeline

How It Works

From scattered data to connected knowledge in three steps.

1

Connect Sources

Add API tokens for GitHub, Discord, or Notion. Point Repollect at your project's public history.

2

Cognify

Data is deduplicated, staged into Cognee, and processed into a rich knowledge graph with entities and relationships.

3

Ask Questions

Chat with an LLM that has full project context. Use slash commands to create issues, fetch PRs, and more.

Capabilities

Core Features

Everything you need to capture and query your project's collective knowledge.

⚙

Multi-Source Ingestion

Ingest from GitHub (issues, PRs, discussions, releases), Discord (channels, threads, messages), Notion (pages, databases, blocks), and local files.

↻

Incremental Sync

Only pulls new or updated data since your last sync. Lightning-fast background jobs keep the graph fresh without re-processing everything.

☼

Auto-Review Scheduler

Background worker continuously polls GitHub for new issues and PRs, fetches historical context from the graph, and posts AI-generated reviews.

✎

Smart Chat Interface

Context-aware chat with specialized slash commands: create issues, fetch PRs, explain diffs, list contributors, and query the knowledge graph.

▢

Two-Phase Ingestion

Phase 1 fetches and stages data (no LLM cost). Phase 2 runs a single cognify() pass for embeddings and graph building.

◆

Deduplication

SHA-256 content hashing ensures no duplicate entries. Same content from different sources is stored once with cross-references.

Quick Start

Get Up and Running

Follow these steps to spin up Repollect locally in under 5 minutes.

# 1. Clone the repository
git clone https://github.com/muskiteer/cognee.git
cd cognee

# 2. Setup the FastAPI Backend
cd app
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 3. Create your .env file
cp .env.example .env # Then add your API keys

# 4. Start the backend
uvicorn main:app --host 0.0.0.0 --port 8000 --reload

# 5. In a new terminal, start the React UI
cd ../ui
npm install
npm run dev
