Chat & Tools

Interact with the knowledge graph through natural language and specialized commands.

Slash Commands

The chat interface supports a range of specialized slash commands that trigger real-time API calls and knowledge graph queries.

CommandDescriptionExample
/issueCreate a new GitHub Issue directly from chat/issue Login button broken on mobile
/NotionCreate a new Notion page/Notion Meeting notes July 4
iss{N}Fetch and discuss a specific GitHub Issueiss54 why was this closed?
pr{N}Fetch and discuss a Pull Requestpr128 what changed here?
diff{N}AI explanation of a PR's code changesdiff128
/contributorsList all contributors with commit counts/contributors
@usernameGet contributor profile and activity summary@muskiteer what have they worked on?

How Chat Works

When a user submits a query, the backend:

  1. Performs a semantic search across the Knowledge Graph via cognee.search()
  2. For specialized slash commands (like pr12), performs a real-time fetch to the GitHub API for live data
  3. Injects both the graph context and live data into the LLM prompt
  4. Returns a grounded response with sources linked
Example: Asking "Why does this project use X instead of Y?" will traverse the graph back to a 3-year-old decision thread, linking code → PR → discussion → person → decision. That chain is what no plain RAG system can do.

Action Tools

Beyond passive chat, Repollect is designed to evolve into an active contributor. Each action tool follows the same pattern: trigger → recall context from graph → LLM decides action → executes via API → reports result.

GitHub Actions

ToolDescription
PR Review AssistantAuto-review PRs using project history context. Finds related issues, past decisions, and flagged regressions.
Issue Triage & LabelingDeduplicate new issues, suggest labels, assign maintainers based on historical expertise.
Context-Aware ReplySuggest draft replies grounded in project history.
Knowledge Gap DetectionIf no graph match for a question, auto-open a GitHub Discussion.
Auto-PR for Known FixesMatch new issues to historical fix patterns, generate draft PRs.
Project Health DashboardAnalyze decision debt, stale PRs, expert locator, knowledge density.
